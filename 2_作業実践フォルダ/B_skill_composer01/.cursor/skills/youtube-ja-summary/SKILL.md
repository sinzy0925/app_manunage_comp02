# YouTube 日本語字幕 高精度要約

入力は **YouTube URL のみ**。字幕テキストはユーザーから受け取らない。
**どんなジャンルの動画でも、事前の変換表・手作業の字幕整形なしで一発実行する。**

必ず本スキルの `scripts/run_pipeline.py` で字幕取得〜10分分割まで行い、その成果物だけから要約する。

## 設計前提（重要）

- パイプラインは **汎用処理のみ**: 字幕取得、VTT掃除（タグ・重複・[音楽]等）、10分分割、軽い候補抽出
- **ジャンル別 ASR 辞書の事前作成・機械一括置換はしない**（任意動画に効かないため）
- 固有名詞・誤変換の直しは、要約時にエージェントが **当該動画の字幕文脈だけで** 確信がある箇所のみ行う
- ユーザー準備物は URL だけ。追加の md / 辞書 / 字幕貼り付けを求めない

## Quick start

作業ディレクトリはリポジトリルート（またはユーザー指定ディレクトリ）。

```powershell
$env:PYTHONIOENCODING = "utf-8"
python .cursor/skills/youtube-ja-summary/scripts/run_pipeline.py "https://www.youtube.com/watch?v=VIDEO_ID"
```

成功時の成果物（`youtube_summary_work/<VIDEO_ID>/`）:

| ファイル | 用途 |
|---|---|
| `meta.json` | URL・タイトル・尺・チャンネル・字幕ソース・開始時刻 |
| `cleaned.txt` | 機械掃除済み全文（タイムスタンプ付き行）。固有名詞は未補正のまま |
| `segments/seg_XX.txt` | 約10分単位の分割テキスト |
| `internal_memo.json` | outline/entities/numeric 候補（ヒューリスティクス。最終MDには出さない） |
| `pipeline_log.txt` | 取得ログ（最終MDには出さない） |

失敗時は要約せず、ログのエラーと利用可能字幕言語を報告して停止する。

## エージェント手順（厳守）

1. **開始時刻**を取得してユーザーに短く報告する  
   `Get-Date -Format "yyyy-MM-dd HH:mm:ss.fff"`（PowerShell）
2. `run_pipeline.py` を実行する（yt-dlp 未導入ならスクリプトが停止するので、その指示どおり報告して終了）
3. `meta.json` と `segments/*.txt` と `internal_memo.json` を読む  
   - Phase A の内部メモは頭の中／メモ用。**最終出力に出さない**
   - 事前辞書ファイルを探したり、ユーザーに用語表を求めたりしない
4. Phase B ルールに従い、**1回で**要約MDを書く（再要約ループ禁止）
5. 出力ファイル名: `YouTube動画要約_<VIDEO_ID>.md`（作業ディレクトリ）
6. **終了時刻**を取得し、開始・終了・所要時間をユーザーへ出力する
7. 出力前に [quality-checks.md](reference/quality-checks.md) を自己確認する

文脈補正の方針: [asr-glossary.md](reference/asr-glossary.md)  
出力雛形: [templates/output.md](templates/output.md)（形式は下記 verbatim を優先）

## 【字幕取得】yt-dlp を必須で使う

- 動画本体はダウンロードしない（`--skip-download`）
- 優先: 日本語手動字幕。なければ日本語自動字幕
- パイプラインが次相当を実行する（手打ち不要）:

```text
yt-dlp --no-update --write-auto-subs --write-subs --sub-lang ja --sub-format vtt --skip-download -o "%(id)s" "{{YOU_TUBE_URL}}"
```

- 生成された `.vtt` を読む
- 取得失敗時は推測で要約せず、エラー内容と利用可能な字幕言語の確認結果を報告して停止する
- 可能なら動画タイトル・尺・チャンネル名も取得してヘッダに使う（任意）

---

## 【Phase A】前処理（clean / enrich 相当。内部処理のみ。途中出力しない）

機械化分は `clean_vtt.py` / `split_segments.py` が実施済み（**表記ゆれ辞書は使っていない**）。
エージェントは成果物を前提に、残りを頭の中で補う。

1. 無視する:
   - タイムスタンプ
   - [音楽][拍手] などのタグ
   - 同一フレーズの連続重複
   - チャンネル登録・宣伝の冗長部

2. 明らかな ASR 誤変換は **この動画の文脈だけで** 修正する:
   - 同一動画内の言い直し・周辺語・タイトル情報で一意になるものだけ直す
   - 英日が1語内で混在している場合は、字幕内で優勢な正規表記の片方に統一
   - **事前のジャンル別変換表は使わない・作らない**
   - 確信が持てない語は原文のまま残し、要約で強調しない

3. 内部メモを作る（最終出力には出さない）:
   - outline_hints: 動画が明示した章立て・「○つの柱」・テーマ見出し
   - entities: 組織・製品・人名とその属性（「誰の何」を混同しないため）
   - numeric_facts: 数値・距離・年・割合など字幕に出た数字
   - uncertain: 年号や数字が欠けていそうな箇所・意味不明な固有名詞

4. 長い動画は約10分単位で区切ってから要約する（コンテキスト切れ防止）  
   → `segments/seg_XX.txt` を順に使う。セグメント数と分割数が一致すること。

---

## 【Phase B】要約ルール（厳守）

- 事実は字幕（と上記内部メモ）のみに基づく
- 字幕にない年号・数値・固有名詞・因果関係は書かない
- uncertain は省略するか「不明」と書く（推測補完しない）
- 別組織・別製品・別人の属性を混ぜない
- 数値・固有名詞は字幕表記を優先し、英日混在表記を作らない
- 宣伝・おまけトークは最後のみ簡潔化、または省略
- 「字幕によると」「セグメントと連続」などのメタ記述は入れない
- 考え順: 当該動画の文脈補正（頭の中）→ 結論 → 章立て要約 → セグメント要約
- 再要約ループはせず、1回で完結する

## 【出力形式】これだけをMDファイルに出力する。取得ログや前置きは出さない

```markdown
================================================================================
YouTube動画 要約
動画URL: {{YOU_TUBE_URL}}
タイトル: （分かれば。不明なら省略可）
チャンネル: （分かれば。不明なら省略可）
動画時間: （分かれば。不明なら「不明」）
字幕ソース: （日本語手動 / 日本語自動生成 を明記）
================================================================================

■ 結論
（全体の主張・核心を短く。箇条書き可）

■ 章立て詳細要約
（テーマ別。outline_hints があればそれに従う。
各章は ■ 見出し + 箇条書きで詳述）

■ 10分単位セグメント要約
--- セグメント00（0:00〜10:00）---
・事実の箇条書き
--- セグメント01（10:00〜20:00）---
・…
（区切った数とセグメント数が一致すること）
```

MD末尾に処理時刻は**書かない**（時刻はチャット側でユーザーへ報告）。

## 【出力前チェック】

- [ ] yt-dlp で字幕を取得済み（未取得なら要約していない）
- [ ] ない年号・数値・固有名詞を補完していない
- [ ] 組織・製品の混同がない
- [ ] 英日混在表記がない
- [ ] メタ文がない
- [ ] セグメント数が10分分割と一致する
- [ ] 開始・終了の時刻を出力した
- [ ] 事前辞書やユーザー追加素材を要求していない
